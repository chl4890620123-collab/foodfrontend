[왜 공통화가 중요한가]

- 특히 "세션 종료/마감/클래스 종료"는 운영 정책과 직접 연결되므로 화면마다 다르면 사용자 혼란이 큽니다.
- 공통 유틸 한 곳에서만 수정하면 Explore/Admin/Detail이 같은 기준으로 동작합니다.

[핵심 유틸 함수 설명]
1) toSessionStatus(session)
- 입력: 세션 객체(정원/예약수/서버 full/completed 값)
- 출력: { full, completed }
- 규칙:
  - full: 서버 full=true 이거나 정원 <= 예약수면 true
  - completed: 서버 completed=true 이거나 시작시각이 현재보다 과거면 true

2) toClassSlotStatus(sessions)
- 입력: 클래스의 전체 세션 배열
- 출력:
  - hasSessions
  - classEnded
  - hasReservableSession
  - amCompleted / pmCompleted
  - amFull / pmFull
- 규칙:
  - 오전/오후 세션을 분리해서 각각 완료/마감 계산
  - 전체 세션이 모두 종료면 classEnded=true
  - 종료 아니고 마감도 아닌 세션이 하나라도 있으면 hasReservableSession=true

3) toIsoDateKey(value)
- 목적: "YYYY-MM-DD" 형태 날짜 키를 통일
- 사용처: 날짜별 세션 필터, 날짜 옵션 생성
